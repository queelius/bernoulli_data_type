A {\ttfamily Regular\+Type} is a concept that requires the following. Consider an object type {\ttfamily X}. If {\ttfamily X} is a {\ttfamily Regular\+Type}, then at minimum it implements assignment, default constructor, copy constructor, and equality comparison.\hypertarget{md_REGULAR_TYPE_CONCEPT_autotoc_md20}{}\doxysection{C++ regular types}\label{md_REGULAR_TYPE_CONCEPT_autotoc_md20}
In C++, a class {\ttfamily X} (object type) that is a {\ttfamily Regular\+Type} if it has the following operations (overload set)\+:


\begin{DoxyEnumerate}
\item Assignment\+: {\ttfamily bool operator=(const X\&)} must be implemented.
\item Default constructor\+: {\ttfamily X\+::\+X()} must be implemented.
\item Copy constructor\+: {\ttfamily X\+::\+X(const X\&)} must be implemented.
\item Equality\+: {\ttfamily bool operator==(const X\&)} must be implemented.
\end{DoxyEnumerate}

For example, the following code snippet must compile successfully.


\begin{DoxyCode}{0}
\DoxyCodeLine{X x;                        // default constructor}
\DoxyCodeLine{X y(x);                     // copy constructor}
\DoxyCodeLine{std::static\_assert(y == x); // equality comparison}
\DoxyCodeLine{y = x;                      // assignment}

\end{DoxyCode}
